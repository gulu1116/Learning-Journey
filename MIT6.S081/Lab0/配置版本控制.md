

## 配置流程

确保已经：

- 安装好 git：`git --version`（没装可以用 `sudo apt install git` 安装）
- 在 GitHub 上注册并登录账号
- 实验环境已经配置完毕（能够成功运行 xv6 并切换 util 分支）
    


### 步骤一：在 GitHub 创建空仓库

1. 登录 [https://github.com](https://github.com/)
2. 点击右上角 “➕” → New repository
3. 名称填写：`xv6-labs-2020`（或者任意名字）
4. 勾选：**Public** 或 **Private**
5. 不勾选 Initialize（即不要创建README）
6. 点击 **Create repository**
    

> 复制 GitHub 提示的仓库地址，例如：

```bash
https://github.com/你的用户名/xv6-labs-2020.git
```


### 步骤二：添加 GitHub 仓库地址为新的远程主机名

当前目录仍然在 `xv6-riscv-riscv/` 中，先看看默认远程地址：

```bash
cat .git/config
```

你会看到如下内容（或类似）：

```ini
[remote "origin"]
    url = git://g.csail.mit.edu/xv6-labs-2020
```

我们不要修改 origin，而是**添加一个新的远程名叫 github**：

```bash
git remote add github https://github.com/你的用户名/xv6-labs-2020.git
```

验证是否添加成功：

```bash
git remote -v
```

你应该看到：

```bash
origin  git://g.csail.mit.edu/xv6-labs-2020 (fetch)
origin  git://g.csail.mit.edu/xv6-labs-2020 (push)
github  https://github.com/你的用户名/xv6-labs-2020.git (fetch)
github  https://github.com/你的用户名/xv6-labs-2020.git (push)
```


### 步骤三：推送代码到 GitHub

**推送某个分支，例如 util：**

```bash
git checkout util     # 切换到 util 分支
git push github util:util
```

这表示：把本地的 `util` 分支推送到 GitHub 的 `util` 分支上。

> 提示输入 GitHub 账户名、密码时，如果你启用了两步验证，请使用 **GitHub token** 作为密码（可在 GitHub 的 Settings → Developer Settings → Personal Access Tokens 创建）


### 步骤四：后续做实验建议（可选但推荐）

**每个实验对应一个测试分支**

```bash
git checkout util                # 切换到主分支
git checkout -b util_test        # 创建并切换到测试分支
```

**写完一个小任务后提交一次代码：**

```bash
git add .
git commit -m "完成 util 实验第一个任务"
```

**完成所有任务后合并回主分支并推送：**

```bash
git checkout util
git merge util_test
git push github util:util
```



### 提示：

|操作|命令|说明|
|---|---|---|
|查看当前分支|`git branch`|有 `*` 标记当前分支|
|切换分支|`git checkout 分支名`|切换到已有分支|
|创建新分支|`git checkout -b 分支名`|创建并切换|
|查看改动文件|`git status`|查看未提交文件|
|查看日志|`git log --oneline`|简洁日志视图|
